    95тЖТ    95тЖТ- тЪая╕П IMPORTANT FINDING for the orchestrator / Agent 2 тАФ TTS FORMAT DEVIATION FROM SPEC:
    96тЖТ    96тЖТ  - The SDK skill doc claimed `response_format: 'mp3'` is supported in non-streaming mode. The LIVE TTS SERVER REJECTS `'mp3'` with HTTP 400 `{"error":{"code":"1214","message":"ф╕НцФпцМБх╜УхЙНresponse_formatхА╝"}}` ("unsupported response_format value"). Verified empirically with `z-ai-web-dev-sdk@0.0.18`.
    97тЖТ    97тЖТ  - Server DOES accept: `response_format: 'wav'` (тЖТ `audio/wav`, RIFF/WAVE), `'pcm'` (тЖТ raw samples), or omitted (тЖТ `audio/pcm`).
    98тЖТ    98тЖТ  - Therefore `audio-synth.ts` requests `response_format: 'wav'` and `SynthResult.format` is typed as `'wav' | 'mp3'` (loosened from the spec's literal `'mp3'`; runtime value is `'wav'`). This is strictly more permissive, so any caller code that does `result.format === 'mp3'` still type-checks.
    99тЖТ    99тЖТ  - The multi-chunk merger (`mergeWavBuffers`) parses each RIFF/WAVE chunk, concatenates the PCM payloads, and rebuilds one canonical 44-byte header тАФ so multi-chunk playback is seamless with correct duration (naive `Buffer.concat` would leave mid-stream RIFF headers some browsers refuse).
